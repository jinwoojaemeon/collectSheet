# 마감지기 프론트엔드 아키텍처 가이드

이 문서는 **현재 `frontend/` 프로젝트의 폴더 구조, API 계층, Zustand, 주요 라이브러리**를 정리하고, **다른 프로젝트에서 동일한 패턴을 재현**할 때 참고할 수 있는 체크리스트를 제공합니다.

협업 시 충돌 최소화 규칙은 [`COLLABORATION_GUIDE_FRONTEND.md`](./COLLABORATION_GUIDE_FRONTEND.md)를 따릅니다.

---

## 1. 기술 스택 요약

| 구분 | 선택 | 비고 |
|------|------|------|
| 빌드/번들 | **Vite 6** | `@vitejs/plugin-react`, `@tailwindcss/vite` |
| UI | **React 18** (peer) | `main.jsx`에서 `createRoot` |
| 스타일 | **Tailwind CSS 4** + **styled-components** + **Emotion** | 전역은 `styles/`, 컴포넌트별는 `*.styled.js` |
| 컴포넌트 UI | **Radix UI** 일괄, **MUI Icons**, **shadcn 스타일** (`app/components/ui/`) | 프로젝트에 따라 일부만 써도 됨 |
| HTTP | **Axios** | 단일 인스턴스 `api/axios.js` |
| 전역 상태 | **Zustand** | `persist`, `subscribeWithSelector` 미들웨어 사용 |
| 라우팅 | **react-router-dom** (의존성 있음) | 메인 앱은 `App.jsx`의 뷰 상태 중심; 일부 레이아웃 컴포넌트에서 `Link`/`useLocation` 사용 가능 |
| 폼 | **react-hook-form** | |
| 지도 | **@vis.gl/react-google-maps** | `main.jsx`에서 `APIProvider`로 선택적 래핑 |
| 실시간 | **@stomp/stompjs** + **sockjs-client** | `services/websocketService.js` |
| 기타 | **sonner** 토스트, **recharts**, **motion**, **date-fns**, **react-dnd** 등 | 도메인 기능에 맞게 선택 |

`package.json`의 전체 의존성 목록이 단일 기준입니다. 새 프로젝트를 만들 때는 **필요한 패키지만** 골라 설치하면 됩니다.

## 2. 소스 디렉터리 구조 (개념도)

```
frontend/
├── index.html
├── vite.config.js
├── package.json
└── src/
    ├── main.jsx                 # 엔트리, 전역 CSS, (선택) Google Maps Provider
    ├── App.jsx                  # 인증 뷰·역할별 대시보드 조합, lazy 로딩, 전역 모달 등
    ├── api/
    │   ├── axios.js             # Axios 인스턴스 + 인터셉터 (토큰, 401/403, refresh)
    │   ├── config.js            # API_BASE_URL, API_TIMEOUT, API_ENDPOINTS, URL 헬퍼
    │   └── services.js          # 도메인별 서비스 객체 + default export 묶음
    ├── store/
    │   ├── authStore.js         # 인증·토큰·persist
    │   ├── chatStore.js         # 채팅 UI + API + (선택) 구독
    │   └── themeStore.js        # 라이트/다크 persist
    ├── contexts/
    │   └── ProjectContext.jsx   # React Context (프로젝트/일정 등 UI 상태)
    ├── hooks/                   # useModal, useChatScroll, useNotificationSource 등
    ├── services/
    │   └── websocketService.js  # STOMP/SockJS 싱글톤 래퍼
    ├── constants/               # 도메인 상수 (예: 프로젝트 회차 상태)
    ├── utils/                   # 날짜 등 순수 유틸
    ├── styles/                  # index.css, tailwind.css, theme, GlobalStyle
    ├── components/
    │   ├── layout/              # FullPageLayout, MainLayout, Header, Sidebar, TopBar …
    │   ├── common/              # 공통 재사용 (Modal, Calendar, AiChatBot …)
    │   └── modals/              # 도메인 공통 모달
    ├── pages/                   # 실제 제품 화면 (권장 작업 위치)
    │   ├── agency/ | artist/ | admin/   # 역할별 하위 폴더
    │   ├── Login/, Signup/, …   # 공개·온보딩
    │   └── …
    └── app/                     # Figma/Make 기반 UI 키트·스타일 가이드 페이지 등
        ├── components/ui/       # Button, Dialog, Card … (Radix + cn)
        └── pages/               # 레거시/데모성 페이지와 병행 존재 가능
```

**역할 분리 요약**

- **`pages/`**: 운영 기능 화면을 **도메인 폴더(`agency/`, `artist/`, `admin/`) 아래**에 두는 패턴이 협업 규칙과 맞습니다.
- **`app/`**: 디자인 시스템·UI 프리미티브·일부 예제 페이지. 새 기능은 가능하면 `pages/` 쪽에 두고, 공통 UI만 `app/components/ui/` 또는 `components/common/`에 추가합니다.

## 3. 엔트리와 앱 셸

### 3.1 `main.jsx`

- `react-dom/client`의 `createRoot`로 마운트.
- 전역 스타일: `./styles/index.css` (그 안에서 tailwind·테마 로드).
- `VITE_GOOGLE_MAPS_API_KEY`가 있으면 `@vis.gl/react-google-maps`의 `APIProvider`로 앱을 감쌉니다.

### 3.2 `App.jsx`

- **역할별 페이지**를 `React.lazy` + `Suspense`로 지연 로딩.
- **Zustand** (`authStore`, `chatStore`), **Context** (`ProjectProvider`), **DndProvider**, **sonner** `Toaster` 등 앱 전역 래퍼.
- OAuth 콜백·URL 쿼리 처리 등은 `useEffect`에서 `window.location` 기준으로 처리하는 패턴이 있습니다.

새 프로젝트에서도 “**엔트리 → 얇은 셸 → lazy 페이지**”만 맞추면 동일한 확장 방식을 유지할 수 있습니다.

## 4. API 계층 (다른 프로젝트에 그대로 옮기기 좋은 패턴)

### 4.1 `api/config.js`

- **`API_BASE_URL`**: `import.meta.env.VITE_API_BASE_URL` (없으면 로컬 백엔드 기본값).
- **`API_TIMEOUT`**: 숫자(ms).
- **`API_ENDPOINTS`**: 경로만 상수/함수로 모음 (REST prefix `/api/...` 일관).
- 업로드 파일·프로필 이미지 URL을 절대 경로로 만드는 **헬퍼 함수** (`getProjectThumbnailUrl`, `getMemberProfileUrl` 등).

**재현 팁**: 엔드포인트는 **도메인별 객체**로 나누고, 새 항목은 **객체 맨 끝에만 추가**해 머지 충돌을 줄입니다.

### 4.2 `api/axios.js`

- `axios.create({ baseURL, timeout, withCredentials, headers })`.
- **요청 인터셉터**: `useAuthStore.getState()`로 `Authorization`, `X-Member-No` 등 부착. `FormData`일 때 `Content-Type` 제거.
- **응답 인터셉터**: `success`/`data` 래핑 규칙 통일, blob은 그대로 반환.
- **401/403**: refresh/reissue 제외 경로에서 `refreshAccessToken` 재시도, 실패 시 `logout` 및 로그인 이동.

### 4.3 `api/services.js`

- 도메인별 **`export const xxxService = { ... }`** (예: `authService`, `memberService`, …).
- 각 메서드는 `api.get/post/put/delete` + `API_ENDPOINTS`만 사용.
- 파일 하단에 **`export default { authService, memberService, ... }`** 로 한 번에 import 가능하게 묶음.

**재현 팁**: named export로 테스트·트리셰이킹에 유리하고, default는 레거시/편의용으로 유지할 수 있습니다.

## 5. Zustand 스토어

| 파일 | 미들웨어 | 주요 책임 |
|------|-----------|-----------|
| `authStore.js` | `persist` | `user`, `token`, `refreshToken`, `login`/`logout`, `refreshAccessToken` (reissue), 영속 키 `auth-storage` 등 |
| `chatStore.js` | `subscribeWithSelector` | 채팅 UI 상태, 방 목록, `chatService`·`websocketService`와 연동 |
| `themeStore.js` | `persist` | `light`/`dark`, `document.documentElement` 클래스 동기화 |

**재현 팁**

- 인증은 **Axios와 동일한 규칙**으로 토큰을 넣을지 정한 뒤, 스토어와 인터셉터를 함께 수정합니다.
- `persist`의 `partialize` 등으로 **민감한 키를 localStorage에 넣지 않도록** 설계할 수 있습니다 (현재 프로젝트는 refresh 등 정책에 맞게 구현됨).

## 6. 그 외 “ glue ” 레이어

| 경로 | 역할 |
|------|------|
| `services/websocketService.js` | STOMP 클라이언트 생명주기, 재연결, 구독 API |
| `contexts/ProjectContext.jsx` | 프로젝트·일정 목업/상태를 여러 화면에서 공유할 때 |
| `hooks/` | 모달·채팅 스크롤·알림 소스 등 UI 로직 재사용 |

## 7. 스타일 전략

- **Tailwind**: 유틸 클래스, `vite.config`에서 `@tailwindcss/vite` 플러그인.
- **styled-components**: 레이아웃·페이지별 `*.styled.js`에서 `styled.div` 등.
- **전역**: `styles/index.css`, `theme.css`, `GlobalStyle.js`(styled-components), 다크 오버라이드 등.
- **MUI/Emotion**: 일부 화면에서 아이콘·컴포넌트 혼용 가능.

새 프로젝트에서는 **“페이지 단위 = styled 파일 + Tailwind 클래스”**만 통일해도 구조는 동일하게 가져갈 수 있습니다.

## 8. Vite 설정 하이라이트 (`vite.config.js`)

- **`@` → `./src`**: import alias.
- **`define.global`**: SockJS 등을 위한 `globalThis` 매핑.
- **`build.rollupOptions.output.manualChunks`**: react, UI, dnd, charts, router 등 분리.
- **`server.proxy['/api']`**: 개발 시 백엔드로 프록시 (로컬 백엔드 포트와 맞출 것).

환경 변수 예:

- `VITE_API_BASE_URL` — API 베이스 URL  
- `VITE_GOOGLE_MAPS_API_KEY` — 지도 사용 시

## 9. 새 프로젝트에서 “동일 구조”로 시작하는 체크리스트

1. **Vite + React** 템플릿 생성 후 `vite.config.js`에 `@` alias, 필요 시 proxy·`global` 정의.
2. **`src/api/`** — `config.js` → `axios.js` → `services.js` 순으로 추가 (엔드포인트·서비스는 끝에만 추가하는 규칙).
3. **`src/store/`** — 인증 스토어 + Axios 인터셉터 연동.
4. **`src/styles/`** — Tailwind 엔트리(`index.css` 등) 한 줄 연결.
5. **`src/pages/`** — `agency` / `artist` / `admin` 등 **도메인 폴더**를 미리 만들고, 페이지는 `페이지명/페이지명Page.jsx`, `페이지명Page.styled.js`, `index.js` 패턴으로 통일.
6. **`src/components/layout`·`common`·`modals`** — 공통 셸과 모달만 분리.
7. **실시간이 필요하면** `services/websocketService.js` + 환경별 WS URL 규칙.
8. **협업 규칙 문서화** — `services.js`/`config.js`/라우트는 “끝에만 추가” 등 [협업 가이드](./COLLABORATION_GUIDE_FRONTEND.md)와 동일하게 팀에 공유.
